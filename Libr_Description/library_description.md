# Описание использования библиотеки

Разработанная библиотека `LibPrimeNum` предоставляет интерфейс для работы со специализированными алгоритмами проверки чисел на простоту (как детерминированными, так и вероятностными), а также генераторами простых чисел на основе сит Эратосфена и Аткина. Библиотека ориентирована на высокую производительность благодаря использованию многопоточности (OpenMP) и поддержке длинной арифметики произвольной точности (GNU MP — GMP).

Для успешной компиляции и интеграции библиотеки в сторонние проекты операционная система должна удовлетворять следующим требованиям:

- **Компилятор:** GCC (с поддержкой стандарта C99/C11 и директив OpenMP).
- **Библиотеки:** * `libgmp` и `libgmp-dev` — для работы с числами произвольной точности.
    - `libpthread` — для обеспечения параллелизма через OpenMP.
    - `libm` — стандартная математическая библиотека (используется для вычисления логарифмов и корней при расчете границ решет).

Библиотека использует внутренний разделяемый контекст — предвычисленную таблицу малых простых чисел размера $512$ элементов (`NUM_PROS_TABL`). Эта таблица критически важна для:

1. Быстрого отсева очевидно составных чисел с помощью пробного деления перед запуском тяжелых тестов (Ферма, Миллера-Рабина, Соловея-Штрассена).
2. Выбора случайных/базовых свидетелей простоты из числа первых простых чисел, что повышает детерминированность тестов на практике.

Поэтому перед вызовом любых функций тестирования простоты **необходимо** выполнить инициализацию, а по завершении работы — освободить ресурсы.

- `void prime_lib_init();` — выделяет память под таблицу малых простых чисел и заполняет её с помощью решета Эратосфена. Вызывается строго один раз в начале работы программы.
- `void prime_lib_cleanup();` — освобождает динамическую память, занятую глобальной таблицей. Вызывается перед завершением работы программы.

Функции проверки на простоту возвращают перечислимый тип данных `PrimeStatus`, который позволяет точно интерпретировать результат:

- `PRIME_COMPOSITE (0)` — число гарантированно является составным.
- `PRIME_PROBABLY (1)` — число с высокой вероятностью является простым (для вероятностных тестов).
- `PRIME_DEFINITELY (2)` — число гарантированно является простым (для детерминированных тестов).

***Интерфейс функций тестирования простоты:***

- **Тест Миллера-Рабина:** `PrimeStatus test_miller_rabin(mpz_t n);`
    Количество итераций (раундов) вычисляется автоматически внутри функции (`optimal_iteration`) на основе битовой длины числа $n$ для достижения оптимального баланса скорости и точности (вероятность ошибки не более $2^{-80}$).
- **Тест Ферма:** `PrimeStatus test_fermat(mpz_t n, int iterations);`
    Параллельный тест. Требует явного указания количества итераций.
- **Тест Соловея-Штрассена:** `PrimeStatus test_solovay_strassen(mpz_t n, int iterations);`
    Использует вычисление символа Якоби. Распараллелен с помощью OpenMP. Требует явного указания количества итераций
- **Тест перебора делителей:** `PrimeStatus test_trivial_division(mpz_t n);`
    Детерминированный тест для малых чисел (эффективен до $10^{12}$).
- **Тест Агравала-Каяна-Саксены (AKS):** `PrimeStatus test_aks(mpz_t n);`
    Строгий полиномиальный детерминированный тест. Из-за высокой вычислительной сложности рекомендуется использовать только для небольших чисел.

***Интерфейс генераторов:***

- **Решето Эратосфена:** `void generate_primes_eratosthenes(int n, unsigned long *out_primes);`
    Заполняет переданный массив `out_primes` первыми `n` простыми числами.
- **Решето Аткина:** `void generate_primes_atkin(int n, unsigned long *out_primes);`
    Альтернативный высокоэффективный генератор на основе квадратичных форм.
